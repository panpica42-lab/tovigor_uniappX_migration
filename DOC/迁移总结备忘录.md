# 迁移总结备忘录

## Coach 组件

老项目 `components/coach` 文件更多，是因为同时维护了 `.vue` 和 `.nvue` 两套实现。

新项目 UniApp X 已收敛为 `.uvue` 单实现：

- `components/coach/coach-box-entry.uvue`：AI 教练气泡统一入口。
- `components/coach/coach-detail-modal.uvue`：AI 教练详情与切换弹窗。

对应关系：

- 老 `coach-box-entry.vue` + `coach-box.vue` + `coach-box-nvue.vue` -> 新 `coach-box-entry.uvue`
- 老 `coach-detail-modal-vue.vue` + `coach-detail-modal-nvue.vue` -> 新 `coach-detail-modal.uvue`

结论：当前不是漏文件，而是把 Vue/NVue 双轨实现合并成 UniApp X 单轨实现。

查漏注意：

- 页面应优先引用 `coach-box-entry.uvue`，不要再恢复老项目的 Vue/NVue 分叉文件。
- 新弹窗接口使用 `visible`、`close`、`select`，后续迁移新页面时不要照搬老接口。
- 老项目智能评估页也用过教练弹窗；如果后续迁智能评估，需要单独接入。
